为了保证 safemask 的极致性能和匹配准确性，请遵循以下规范：
1. 正则编写准则 (Principles)
脱敏而非校验：我们的目的是遮盖隐私，不是验证数据的合法性。宁可错杀（遮盖范围稍大），不可漏过。 复杂的正则（如严格的 Email 校验）会显著降低匹配速度。
词边界优先：始终在正则两端使用 \b。这能防止把 ID_12345678901 中的数字误认为手机号。
禁用锚点：严禁在 rules 中使用 ^ 和 $（除非你确定要匹配整行）。
禁止环视：Rust regex 库不支持 (?!)、(?<=) 等。如果需要排除，请通过字符集 [^...] 实现。

group: "CATEGORY_NAME" # 大类名称（如 AUTH, PII, NETWORK）
rules:
  - name: "Rule_Unique_Name" # 规则描述名
    # 模式建议：
    # 1. 使用单引号包裹，防止 YAML 转义冲突
    # 2. 模式开头结尾尽量使用 \b
    # 3. 避免过度使用 (.*) 等贪婪匹配
    pattern: '\bPREFIX-[a-zA-Z0-9]{10,}\b'
    mask: "<LABEL>" # 建议使用大写+尖括号，AI 识别效果最好
