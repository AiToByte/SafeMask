通过对输入明文与脱敏输出的深度比对，我们可以诊断出 SafeMask 目前在混合引擎（规则+AI）协同工作时，存在几个底层的**硬伤与漏洞**。这些问题导致了严重的标签错位、信息吞噬和误伤。

以下是针对这些问题的深度病因分析，并结合前沿技术给出的高质量重构与迭代方案。

---

### 一、 现状病因诊断（Root Cause Analysis）

#### 1. 灾难性的字符与字节偏移量不匹配（Character vs. Byte Offset Mismatch）
这是导致脱敏文末尾出现 `[BANK_CARD]，[BANK_CARD][PERSON][BANK_CARD]。` 以及中途出现 `https[URL]angsan.<DOMAIN>/about` 乱码的**根本原因**。

* **原理分析**：
  * **Rust 字符边界**：Rust 的 `String` 和 `&[u8]` 字节数组在进行切片（Slice）操作时，使用的是**字节偏移量（Byte Offset）**。在 UTF-8 编码下，一个英文字符占 1 字节，而一个中文字符通常占 3 字节（如“你”占 3 字节）。
  * **HuggingFace 分词器行为**：`tokenizers` 库在对文本进行 `encode` 并通过 `get_offsets()` 返回 Token 跨度时，默认返回的是**字符偏移量（Character Offset，即 Unicode 字符索引）**，或者是 UTF-32 编码下的字符位置。
* **病因推导**：
  在原文中，前半部分含有大量中文。当 AI 识别到后面的实体时（例如识别到“张三”），分词器返回的字符偏移量（例如 `char_start = 50`）被 Rust 引擎**直接当作字节偏移量** `byte_start = 50` 使用。
  由于中文字符的字节数大于字符数，`byte_start = 50` 对应的物理位置其实远远落在了真实的“张三”之前（大约在文本的中前部）。这导致脱敏标签被强行插入到了错误的字节区间，不仅破坏了邻近的 UTF-8 字符边界，还在文本末尾留下了大量未被正确替换的残存字符。

#### 2. 正则边界锚定失效导致的“子串溢出”（Sub-string Bleeding）
观察这一句：`我的银行卡是中国建设银行 6227<GLOBAL_PHONE>。`
* **原因分析**：
  内置规则 `Global_Phone_E164` 的正则表达式为 `(?:\+|00)[1-9]\d{7,14}\b`。
  该正则在末尾使用了 `\b`（词边界），但**在开头没有使用 `\b` 或者是边界锚定**。
  在银行卡号 `6227001234560012345` 中，子串 `001234560012345` 恰好满足以 `00` 开头、后续跟非零数字、再跟 13 位数字的模式。由于正则左侧没有边界限制，它直接在数字内部发生匹配，将银行卡号的后半部分误判为了全球电话，导致了极其严重的误伤。

#### 3. 粗暴的“整块丢弃”冲突消解算法
现有的 `ConflictResolver` 在面对重叠区间时，采用的是“保留高优先级，直接丢弃重叠的低优先级区间”的策略。
* **原因分析**：
  当 AI 识别出地址 `[ADDRESS]` 为 `"北京市海淀区中关村大街1号院2号楼3单元401室"`，而规则引擎识别出其中的 `"1"`、`"2"`、`"3"`、`"401"` 为数字或其它标签时，如果规则引擎的优先级高于 AI，AI 的整个地址标签就会被**直接丢弃**。
  反之，如果 AI 优先级高，则整个大区间被替换为 `[ADDRESS]`，从而吞噬了其中本可以被正则精确提取的特定子实体。这种非黑即白的吞噬，导致了脱敏流的精细度受限。

### 二、 前沿理论与解决方案（State-of-the-Art Solutions）

为了实现工业级、零误差的脱敏效果，我们从**字符编码学、区间几何学（Interval Geometry）以及防御性编译**三个维度引入以下设计：

#### 1. 精密的 Unicode 双向偏移投射器（Bi-directional Offset Realignment）
解决 character-to-byte 转换的科学方案是：在推理前，为输入文本构建一个 $O(1)$ 时间复杂度的**字节-字符索引映射表（Offset Mapping Table）**。
在执行后处理时，通过投射器将分词器的字符位置无损转换为 Rust 物理字节切片位置，彻底解决标签错位和乱码。

#### 2. 子区间“雕刻”与非破坏性分割算法（Sub-span Carving Algorithm）
借鉴区间几何学，当一个低优先级大区间（如 AI 识别的 `[ADDRESS]`：0..40）与一个高优先级小区间（如正则识别的 `[IP]`：15..25）重叠时，**不应丢弃任何一个**。
我们应采用**“雕刻（Carving）”算法**，将大区间沿高优先级边界切开，分裂为 `0..15`（ADDRESS）和 `25..40`（ADDRESS），中间保留 `15..25`（IP）。

```
原始区间 (AI):   [─────────────────── ADDRESS (0..40) ───────────────────]
高优区间 (Regex):                [── IP (15..25) ──]
冲突消解后:      [─ ADDRESS (0..15) ─][── IP (15..25) ──][─ ADDRESS (25..40) ─]
```

#### 3. 防御性正则重构（Defensive Regex Anchoring）
对于所有纯数字或可能在长数字内部发生部分匹配的模式，强制使用**双侧词边界（`\b`）**或**排除性字符集**。在不支持正向/反向预查（Lookaround）的 Rust `regex` 库中，通过显式声明左侧边界来防范子串溢出。

### 三、 核心代码重构方案（Refactoring Code）

#### 1. 重构 AI 引擎：引入字符到字节投射器
修改 `src-tauri/src/infra/ai/ner_engine.rs` 中的推理和解码逻辑。在执行 BIOES 解码前，先计算原文本的字符字节投射表：

```rust
// src-tauri/src/infra/ai/ner_engine.rs

/// 🚀 新增：构建字符索引到字节索引的映射表
/// 能够在 O(1) 复杂度内将分词器的字符 offset 转换为 Rust 物理字节 offset
fn build_char_to_byte_map(text: &str) -> Vec<usize> {
    let mut map = Vec::with_capacity(text.chars().count() + 1);
    for (byte_idx, _) in text.char_indices() {
        map.push(byte_idx);
    }
    map.push(text.len()); // 放入哨兵值，处理边界情况
    map
}

impl NerEngine {
    pub fn infer(&mut self, text: &str) -> Result<Vec<EntitySpan>> {
        if text.trim().is_empty() {
            return Ok(Vec::new());
        }

        // 1. 构建物理字节映射表
        let char_to_byte = build_char_to_byte_map(text);

        // 2. Tokenize
        let encoding = self.tokenizer.encode(text, false)
            .map_err(|e| anyhow::anyhow!("Tokenize 失败: {}", e))?;

        let token_ids: Vec<u32> = encoding.get_ids().to_vec();
        let attention_mask: Vec<u32> = encoding.get_attention_mask().to_vec();
        
        // 🚀 核心修正：分词器返回的 offsets 是字符单位的（char-based）
        let char_offsets: Vec<(usize, usize)> = encoding.get_offsets().to_vec();
        let seq_len = token_ids.len().min(self.max_length);

        let input_ids_i64: Vec<i64> = token_ids[..seq_len].iter().map(|&x| x as i64).collect();
        let attention_mask_i64: Vec<i64> = attention_mask[..seq_len].iter().map(|&x| x as i64).collect();

        // 3. 转换为物理字节 offsets
        let mut byte_offsets = Vec::with_capacity(seq_len);
        for &(c_start, c_end) in &char_offsets[..seq_len] {
            // 通过映射表进行物理投射
            let b_start = *char_to_byte.get(c_start).unwrap_or(&text.len());
            let b_end = *char_to_byte.get(c_end).unwrap_or(&text.len());
            byte_offsets.push((b_start, b_end));
        }

        // 4. 执行推理 (ort 2.0 API)
        let (seq_len_actual, num_labels, logits_owned) = {
            let input_ids_tensor = Tensor::from_array(([1, seq_len], input_ids_i64))?;;
            let attention_mask_tensor = Tensor::from_array(([1, seq_len], attention_mask_i64))?;

            let outputs = self.session.run(ort::inputs![
                "input_ids" => input_ids_tensor,
                "attention_mask" => attention_mask_tensor,
            ])?;

            let logits_value = outputs.get("logits").context("未找到 logits")?;
            let (shape, logits_slice) = logits_value.try_extract_tensor::<f32>()?;

            (shape[1] as usize, shape[2] as usize, logits_slice.to_vec())
        };

        let logits_view = ndarray::ArrayView3::from_shape(
            (1, seq_len_actual, num_labels),
            &logits_owned,
        )?;

        // 🚀 传入转换好的 byte_offsets 替换原有的 char_offsets
        let spans = self.decode_bioes(&logits_view, &byte_offsets, text, seq_len_actual);

        Ok(spans)
    }
}
```

#### 2. 重构冲突解决层：实现非破坏性子区间“雕刻”算法
现有的 `ConflictResolver`（位于 `src-tauri/src/core/resolver/mod.rs`）需要从“简单丢弃”升级为“区间分裂与雕刻”。通过对冲突区间的几何级重排，保证高、低优先级特征共存。

请更新冲突解决算法实现：

```rust
// src-tauri/src/core/resolver/mod.rs

use crate::core::recognizer::EntitySpan;
use log::debug;

pub struct ConflictResolver {
    confidence_threshold: f32,
}

impl ConflictResolver {
    pub fn new(confidence_threshold: f32) -> Self {
        Self {
            confidence_threshold: confidence_threshold.clamp(0.0, 1.0),
        }
    }

    /// 🚀 核心算法：子区间雕刻合并（Sub-span Carving）
    pub fn resolve(&self, spans: Vec<EntitySpan>) -> Vec<EntitySpan> {
        if spans.is_empty() {
            return spans;
        }

        // 1. 过滤低置信度结果
        let mut candidates: Vec<EntitySpan> = spans
            .into_iter()
            .filter(|s| s.confidence >= self.confidence_threshold)
            .collect();

        // 2. 核心排序：高优先级（priority）在最前；若优先级相同，则宽区间（Length）在最前
        candidates.sort_by(|a, b| {
            b.confidence.partial_cmp(&a.confidence).unwrap_or(std::cmp::Ordering::Equal)
                .then(b.priority().cmp(&a.priority()))
                .then((b.end - b.start).cmp(&(a.end - a.start)))
        });

        let mut accepted_spans: Vec<EntitySpan> = Vec::new();

        // 3. 逐个将候选区间“雕刻”并注入到已接收列表中
        for mut candidate in candidates {
            let mut fragments = vec![candidate];

            // 用所有已经接受的高优先级区间，去裁剪当前候选区间
            for accepted in &accepted_spans {
                let mut next_fragments = Vec::new();

                for frag in fragments {
                    if !frag.overlaps_with(accepted) {
                        // 无重叠，保留当前碎片
                        next_fragments.push(frag);
                    } else {
                        // 发生重叠，执行区间雕刻（Carving）
                        
                        // 情况 A：Accepted 完全吞噬了 Frag (Accepted 包含了 Frag)
                        if accepted.start <= frag.start && accepted.end >= frag.end {
                            continue; // 碎片完全被吞噬，不保留
                        }

                        // 情况 B：Accepted 处于 Frag 内部 (Frag 包含了 Accepted)，分裂为左右两个碎片
                        if accepted.start > frag.start && accepted.end < frag.end {
                            let mut left = frag.clone();
                            left.end = accepted.start;

                            let mut right = frag.clone();
                            right.start = accepted.end;

                            next_fragments.push(left);
                            next_fragments.push(right);
                            continue;
                        }

                        // 情况 C：Accepted 与 Frag 左重叠 (Accepted 遮盖了 Frag 的左半部分)
                        if accepted.start <= frag.start && accepted.end < frag.end {
                            let mut right = frag.clone();
                            right.start = accepted.end;
                            next_fragments.push(right);
                            continue;
                        }

                        // 情况 D：Accepted 与 Frag 右重叠 (Accepted 遮盖了 Frag 的右半部分)
                        if accepted.start > frag.start && accepted.end >= frag.end {
                            let mut left = frag.clone();
                            left.end = accepted.start;
                            next_fragments.push(left);
                            continue;
                        }
                    }
                }
                fragments = next_fragments;
            }

            // 收集所有雕刻后幸存的无冲突碎片
            accepted_spans.extend(fragments);
        }

        // 4. 最终输出：按起始位置升序排列，便于脱敏引擎顺序替换
        accepted_spans.sort_by_key(|s| s.start);
        accepted_spans
    }
}
```
