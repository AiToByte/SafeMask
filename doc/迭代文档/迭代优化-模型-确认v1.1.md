## 一、 📌 待确认项最佳工程实践分析 (Q1 - Q5)

### Q1: 模型发布形式 — 单归档 vs 多文件直链
* **可行性建议：选择 A 方案（单 zip 包 `privacy-filter.zip`）**。
* **原因分析**：
  * **事务一致性**：逐个文件下载（B 方案）在网络不稳定时极易发生“下到第三个文件断网，留下前两个文件破坏状态”的尴尬局面，状态机管理异常复杂。而 Zip 压缩包具有**事务性**，要么下载完全部解压，要么直接报错，逻辑极其纯粹。
  * **网络开销**：Zip 的压缩算法（Deflate）对于文本类的 `tokenizer.json`、`config.json` 以及 ONNX 结构文件有极高的压缩比，能显著缩短下载体积。
  * **解压时状态反馈**：下载到 100% 后，前端展示 `解压中 (Extracting...)`，解压完成后直接置为 `READY`。

### Q2: 下载 URL 与 Release tag
* **可行性建议**：
  * **发布方案**：在 GitHub Release `v1.2.4`（或最新正式 tag）下上传 `privacy-filter.zip`。
  * **国内网络加速（极重要）**：GitHub Release 的大文件在亚洲地区常常由于 Fastly 节点被限速或丢包，900MB 的文件直接从 GitHub 下载大概率会发生中途中断。
  * **完美方案**：采用 **双 URL 轮询策略**。国内环境优先尝试**国内镜像或免费托管云（如 ModelScope 魔搭社区、Hugging Face Mirror 镜像站）**，若请求失败，则自动降级回滚至 **GitHub Release 原生链接**。

### Q3: 是否保留便携版手动放置路径
* **可行性建议：绝对需要保留**。
* **原因分析**：
  * SafeMask 作为一款主打极客和隐私的工具，支持“便携版（绿色版）”是极好的加分项。
  * 保持您设计的路径查找优先级（1. `resource_dir` 便携场景 -> 2. `app_local_data_dir` 正常下载场景 -> 3. Fallback）是标准的专业软件设计逻辑。

### Q4: `ort` crate 的 `copy-dylibs` feature
* **可行性建议**：
  * 是的，必须在 `Cargo.toml` 中配置 `ort = { version = "...", features = ["copy-dylibs"] }`。
  * 在打包（`build`）时，`ort` 编译脚本会自动将依赖的动态链接库（`onnxruntime.dll`/`libonnxruntime.dylib`）拷贝到编译输出目录（如 `target/release/`）。

### Q5: 安装后 ONNX Runtime DLL 缺失
* **可行性建议**：
  * 此问题确实普遍存在。如果在没有安装 ONNX 环境的干净 Windows 机器上直接运行打包后的 SafeMask，会由于系统找不到 `onnxruntime.dll` 导致 AI 模块报错或静默闪退。
  * **彻底解决措施**：在 `tauri.conf.json` 的 `bundle -> resources` 中，不仅要配置模型，还要显式将刚才编译生成的 `.dll` / `.dylib` 库文件注册为应用资源，确保它们随安装包分发并在运行时通过系统的 `LD_LIBRARY_PATH` 或本地可执行文件同级路径被识别。

/// 检查模型文件完整性 (四件套校验)
#[tauri::command]
pub fn check_model_file(app: AppHandle) -> Result<String, String> {
    // 优先尝试便携式路径 (resource_dir)
    let portable_dir = app.path().resource_dir().unwrap_or_default().join("models").join("privacy-filter");
    if verify_model_integrity(&portable_dir) {
        return Ok("READY_PORTABLE".to_string());
    }

    // 其次尝试本地数据路径 (app_local_data_dir)
    let local_dir = app.path().app_local_data_dir().unwrap_or_default().join("models").join("privacy-filter");
    if verify_model_integrity(&local_dir) {
        return Ok("READY".to_string());
    }

    Ok("MISSING".to_string())
}

/// 内部函数：验证4个核心模型组件是否健全
fn verify_model_integrity(dir: &Path) -> bool {
    let files = ["model_q4.onnx", "model_q4.onnx_data", "tokenizer.json", "config.json"];
    for file in &files {
        let path = dir.join(file);
        if !path.exists() {
            return false;
        }
        // 粗略大小验证，防止残缺空文件
        if let Ok(meta) = std::fs::metadata(&path) {
            if meta.len() == 0 {
                return false;
            }
        }
    }
    true
}

/// 启动下载与解压流水线
#[tauri::command]
pub async fn start_model_download(
    app: AppHandle,
    state: tauri::State<'_, DownloadState>,
    url: String,
) -> Result<String, String> {
    state.cancel_flag.store(false, Ordering::SeqCst);
    let cancel_token = state.cancel_flag.clone();

    let local_data = app.path().app_local_data_dir().map_err(|e| e.to_string())?;
    let model_dir = local_data.join("models").join("privacy-filter");
    tokio::fs::create_dir_all(&model_dir).await.map_err(|e| e.to_string())?;

    // 🚀 核心看守：检测磁盘剩余空间是否大于 2.2GB (压缩包 550M + 解压后 950M)
    if let Ok(available_space) = fs2::available_space(&model_dir) {
        let required_space = 2 * 1024 * 1024 * 1024; // 2 GB
        if available_space < required_space {
            return Err("DISK_SPACE_LOW".to_string());
        }
    }

    let zip_path = model_dir.join("privacy-filter.zip");
    let app_clone = app.clone();

    tokio::spawn(async move {
        match perform_download_pipeline(&app_clone, &url, &zip_path, &model_dir, cancel_token).await {
            Ok(_) => {
                let _ = app_clone.emit("model-download-status", "READY");
            }
            Err(e) => {
                let _ = app_clone.emit("model-download-status", format!("ERROR: {}", e));
            }
        }
    });

    Ok("STARTED".to_string())
}

#[tauri::command]
pub fn cancel_model_download(state: tauri::State<'_, DownloadState>) -> Result<(), String> {
    state.cancel_flag.store(true, Ordering::SeqCst);
    Ok(())
}

async fn perform_download_pipeline(
    app: &AppHandle,
    url: &str,
    zip_path: &Path,
    extract_to: &Path,
    cancel_token: Arc<AtomicBool>,
) -> Result<(), String> {
    let client = reqwest::Client::builder()
        .timeout(std::time::Duration::from_secs(30))
        .build()
        .map_err(|e| e.to_string())?;

    let res = client.get(url).send().await.map_err(|e| format!("网络连接超时，请重试: {}", e))?;
    if !res.status().is_success() {
        return Err(format!("HTTP 状态码错误: {}", res.status()));
    }

    let total_size = res.content_length().unwrap_or(0);
    let mut file = File::create(zip_path).await.map_err(|e| format!("写入文件失败: {}", e))?;
    let mut stream = res.bytes_stream();
    
    let mut downloaded: u64 = 0;
    let start_time = Instant::now();
    let mut last_emit = Instant::now();

    while let Some(chunk_result) = stream.next().await {
        if cancel_token.load(Ordering::SeqCst) {
            drop(file);
            let _ = tokio::fs::remove_file(zip_path).await;
            return Err("DOWNLOAD_CANCELLED".to_string());
        }

        let chunk = chunk_result.map_err(|e| format!("数据流读取中断: {}", e))?;
        file.write_all(&chunk).await.map_err(|e| format!("磁盘写入失败: {}", e))?;
        downloaded += chunk.len() as u64;

        if last_emit.elapsed().as_millis() >= 100 || downloaded == total_size {
            let elapsed = start_time.elapsed().as_secs_f64();
            let speed_mbps = if elapsed > 0.0 {
                (downloaded as f64 / (1024.0 * 1024.0)) / elapsed
            } else {
                0.0
            };
            let percentage = (downloaded as f64 / total_size as f64) * 100.0;

            let _ = app.emit("model-download-progress", ProgressPayload {
                percentage,
                downloaded_bytes: downloaded,
                total_bytes: total_size,
                speed_mbps,
            });
            last_emit = Instant::now();
        }
    }

    file.flush().await.map_err(|e| e.to_string())?;
    drop(file);

    // 🚀 下载完成，转换状态到前端：EXTRACTING（解压中）
    let _ = app.emit("model-download-status", "EXTRACTING");

    // 阻塞/解压任务交给标准线程池，不占用异步响应
    let zip_path_buf = zip_path.to_path_buf();
    let extract_to_buf = extract_to.to_path_buf();
    
    tokio::task::spawn_blocking(move || {
        extract_zip(&zip_path_buf, &extract_to_buf)
    }).await.map_err(|e| e.to_string())??;

    // 解压成功，彻底删除临时 zip 包
    let _ = std::fs::remove_file(zip_path);

    Ok(())
}

/// 同步解压函数
fn extract_zip(zip_path: &Path, extract_to: &Path) -> Result<(), String> {
    let file = std::fs::File::open(zip_path).map_err(|e| e.to_string())?;
    let mut archive = zip::ZipArchive::new(file).map_err(|e| e.to_string())?;

    for i in 0..archive.len() {
        let mut file = archive.by_index(i).map_err(|e| e.to_string())?;
        let outpath = match file.enclosed_name() {
            Some(path) => extract_to.join(path),
            None => continue,
        };

        if (*file.name()).ends_with('/') {
            std::fs::create_dir_all(&outpath).map_err(|e| e.to_string())?;
        } else {
            if let Some(p) = outpath.parent() {
                if !p.exists() {
                    std::fs::create_dir_all(p).map_err(|e| e.to_string())?;
                }
            }
            let mut outfile = std::fs::File::create(&outpath).map_err(|e| e.to_string())?;
            std::io::copy(&mut file, &mut outfile).map_err(|e| e.to_string())?;
        }
    }
    Ok(())
}

#### 2.1 完善前端状态机逻辑 `src/hooks/useModelDownloader.ts`
支持 `EXTRACTING` 和磁盘空间不足的错误提示：

```typescript
// src/hooks/useModelDownloader.ts
import { create } from "zustand";
import { invoke } from "@tauri-apps/api/core";
import { listen, type UnlistenFn } from "@tauri-apps/api/event";

export type DownloadStatus = "IDLE" | "CHECKING" | "MISSING" | "DOWNLOADING" | "EXTRACTING" | "READY" | "ERROR";

interface ProgressData {
  percentage: number;
  downloaded_bytes: number;
  total_bytes: number;
  speed_mbps: number;
}

interface ModelDownloaderState {
  status: DownloadStatus;
  progress: ProgressData;
  errorMessage: string | null;
  etaSeconds: number;
  checkStatus: () => Promise<DownloadStatus>;
  startDownload: (url: string) => Promise<void>;
  cancelDownload: () => Promise<void>;
}

export const useModelDownloader = create<ModelDownloaderState>((set, get) => {
  let unlistenProgress: UnlistenFn | null = null;
  let unlistenStatus: UnlistenFn | null = null;

  const cleanup = () => {
    if (unlistenProgress) { unlistenProgress(); unlistenProgress = null; }
    if (unlistenStatus) { unlistenStatus(); unlistenStatus = null; }
  };

  return {
    status: "IDLE",
    progress: { percentage: 0, downloaded_bytes: 0, total_bytes: 0, speed_mbps: 0 },
    errorMessage: null,
    etaSeconds: -1,

    checkStatus: async () => {
      set({ status: "CHECKING" });
      try {
        const res = await invoke<string>("check_model_file");
        // 兼容便携版状态
        const isReady = res === "READY" || res === "READY_PORTABLE";
        set({ status: isReady ? "READY" : "MISSING", errorMessage: null });
        return isReady ? "READY" : "MISSING";
      } catch (e: any) {
        set({ status: "ERROR", errorMessage: e });
        return "ERROR";
      }
    },

    startDownload: async (url: string) => {
      if (get().status === "DOWNLOADING" || get().status === "EXTRACTING") return;
      cleanup();
      
      set({
        status: "DOWNLOADING",
        progress: { percentage: 0, downloaded_bytes: 0, total_bytes: 0, speed_mbps: 0 },
        errorMessage: null,
        etaSeconds: -1,
      });

      unlistenProgress = await listen<ProgressData>("model-download-progress", (e) => {
        const remaining = e.payload.total_bytes - e.payload.downloaded_bytes;
        const speed = e.payload.speed_mbps * 1024 * 1024;
        const eta = speed > 0 ? Math.ceil(remaining / speed) : -1;
        set({ progress: e.payload, etaSeconds: eta });
      });

      unlistenStatus = await listen<string>("model-download-status", (e) => {
        const signal = e.payload;
        if (signal === "READY") {
          set({ status: "READY", errorMessage: null });
          cleanup();
        } else if (signal === "EXTRACTING") {
          set({ status: "EXTRACTING" });
        } else if (signal.startsWith("ERROR:")) {
          const rawErr = signal.replace("ERROR:", "").trim();
          const isCancelled = rawErr === "DOWNLOAD_CANCELLED";
          set({
            status: isCancelled ? "MISSING" : "ERROR",
            errorMessage: isCancelled ? null : rawErr,
          });
          cleanup();
        }
      });

      try {
        await invoke("start_model_download", { url });
      } catch (e: any) {
        if (e === "DISK_SPACE_LOW") {
          set({ status: "ERROR", errorMessage: "用户本地磁盘空间不足 (至少需要 2.2GB 剩余空间)" });
        } else {
          set({ status: "ERROR", errorMessage: e });
        }
        cleanup();
      }
    },

    cancelDownload: async () => {
      try {
        await invoke("cancel_model_download");
      } catch (e: any) {
        set({ errorMessage: `中断失败: ${e}` });
      }
    },
  };
});
```

                </span>
                <span className="text-amber-400 font-extrabold">{progress.percentage.toFixed(1)}%</span>
              </div>
              <div className="relative w-full h-2 bg-black/60 rounded-full border border-white/[0.03] shadow-inner overflow-hidden">
                <div
                  className="h-full bg-gradient-to-r from-amber-500 to-amber-400 rounded-full shadow-[0_0_12px_#f59e0b] transition-all duration-150"
                  style={{ width: `${progress.percentage}%` }}
                />
              </div>
            </div>
            <div className="flex justify-between items-center text-xs font-mono text-zinc-500">
              <div className="flex gap-4">
                <span>速度: <strong className="text-zinc-300 font-bold">{progress.speed_mbps.toFixed(1)} MB/s</strong></span>
                <span>预计剩余: <strong className="text-zinc-300 font-bold">{etaSeconds < 0 ? "估算中..." : `${etaSeconds}s`}</strong></span>
              </div>
              <button
                type="button"
                onClick={() => { cancelDownload(); play("CLICK"); }}
                className="flex items-center gap-1.5 px-3 py-1.5 rounded-lg border border-red-500/20 bg-red-500/5 text-red-400 hover:bg-red-500/10 transition-all text-[10px]"
              >
                <X size={10} /> 中断
              </button>
            </div>
          </div>
        )}

        {status === "EXTRACTING" && (
          <div className="flex flex-col items-center justify-center py-6 gap-3 animate-pulse">
            <Loader2 size={24} className="text-blue-400 animate-spin" />
            <div className="text-center">
              <p className="text-xs font-bold text-zinc-300">正在执行原子解压与完整性校验...</p>
              <p className="text-[10px] text-zinc-600 mt-1">这大约需要 5 - 15 秒，请不要关闭程序</p>
            </div>
          </div>
        )}

        {status === "ERROR" && (
          <div className="space-y-4">
            <div className="flex items-start gap-3 p-4 rounded-2xl bg-red-500/[0.02] border border-red-500/15">
              <AlertTriangle size={16} className="text-red-400 shrink-0 mt-0.5" />
              <div>
                <p className="text-xs text-red-400 font-bold">加载任务中断</p>
                <p className="text-[11px] text-zinc-600 mt-1 leading-relaxed">
                  原因: {errorMessage || "请求超时，请检查网络或代理配置。"}
                </p>
              </div>
            </div>
            <button
              type="button"
              onClick={() => { startDownload(DOMESTIC_FAST_URL); play("CLICK"); }}
              className="w-full py-4 bg-zinc-900 border border-white/5 text-zinc-400 hover:text-amber-200 hover:border-amber-500/20 font-bold rounded-2xl text-xs uppercase tracking-widest transition-all"
            >
              <RefreshCw size={12} className="inline mr-2" /> 重新尝试连接
            </button>
          </div>
        )}
      </div>
    </div>
  );
}

## 三、 ⚠️ 补充风险与应对预案

| 风险 | 影响 | 精细化缓解措施 |
| :--- | :--- | :--- |
| **中途解压磁盘爆满** | 解压线程中途写入失败导致崩溃 | 在 Rust 解压流 `extract_zip` 的循环中，使用 `fs2` **每解压一个子文件前检查一次可用空间**，剩余不足 100MB 自动触发解压中断，清理残余，提示用户。 |
| **ONNX Runtime 缺失** | 导致 AI 引擎在加载时触发静态链接或动态库缺失异常 | 将 `onnxruntime.dll` / `dylib` 显式写入打包。若发布便携版，则提供包含依赖库的 Zip 包。 |
| **网络波动频繁重试** | 请求卡死在 Half-Open 状态 | 显式配置 `reqwest::Client` 的 `timeout(30s)` 及 `pool_idle_timeout`，保障每次断网后能在 30 秒内迅速切断并安全抛出错误，而不是静默死锁。 |
