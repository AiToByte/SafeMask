# SafeMask 鉴权代理服务器操作速查清单

## 上传到服务器

将以下 **4 个文件** 上传到服务器的同一目录（例如 `/opt/safemask-auth/`）：

```bash
scp workers/download-proxy/server.js            root@你的IP:/opt/safemask-auth/
scp workers/download-proxy/init-key.sh          root@你的IP:/opt/safemask-auth/
scp workers/download-proxy/deploy.sh            root@你的IP:/opt/safemask-auth/
scp workers/download-proxy/nginx.conf.template  root@你的IP:/opt/safemask-auth/
```

然后在服务器上设为可执行：

```bash
chmod +x /opt/safemask-auth/*.sh
```

## 一键部署（deploy.sh）

```bash
cd /opt/safemask-auth
sudo bash deploy.sh \
  --domain www.xiao.org.cn \
  --download-url https://obs.be.com:9004/gxzh/2026/07/06/privacy-filter.zip
```

deploy.sh 会自动执行：
1. 若 HMAC 密钥不存在 → 调用 `init-key.sh` 自动生成并保存
2. 若密钥已存在 → 直接复用
3. 检查前置条件（Node.js ≥ 18, openssl, PM2）
4. 通过 PM2 启动/重启鉴权服务（注入 `HMAC_KEY` + `DOWNLOAD_URL`）
5. 运行 6 项 curl 验证（健康检查 / 无 token / 无效路径 / 无效 token / 有效 token 302 / 频率限制 429）
6. 输出部署摘要

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `--domain` | **必填** 鉴权服务域名 | `www.xiao.org.cn` |
| `--download-url` | **必填** OSS 真实下载地址 | `https://obs.be.com:9004/...` |
| `--key-dir` | 密钥存储目录（默认 `/opt/safemask-auth/config`） | |
| `--config-dir` | server.js 所在目录（默认 `/opt/safemask-auth`） | |
| `--port` | Node.js 监听端口（默认 3000） | |
| `--dry-run` | 仅打印操作，不实际执行 | |
| `--restart-only` | 仅重启 PM2，跳过 curl 验证 | |
| `--skip-verify` | 跳过部署后的 curl 验证 | |

### 子命令

```bash
# 仅重启鉴权服务
sudo bash deploy.sh --restart-only

# 模拟运行（查看所有操作但不执行）
sudo bash deploy.sh \
  --domain www.xiao.org.cn \
  --download-url https://obs.be.com:9004/gxzh/2026/07/06/privacy-filter.zip \
  --dry-run
```

## Nginx HTTPS 反向代理（手动配置）

脚本**不管理 Nginx**，以下操作只需首次部署时执行一次。

### 1. 配置参考

参考 `nginx.conf.template` 文件，将 `{{SERVER_DOMAIN}}` 替换为你的域名：

```nginx
server {
    listen 443 ssl http2;
    server_name www.xiao.org.cn;          # ← 替换为实际域名

    ssl_certificate     /etc/letsencrypt/live/www.xiao.org.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/www.xiao.org.cn/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    location / {
        proxy_pass http://127.0.0.1:3000;  # ← 与 --port 参数一致
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name www.xiao.org.cn;
    return 301 https://$server_name$request_uri;
}
```

### 2. 启用

```bash
# 写入 /etc/nginx/sites-available/safemask-auth
# 然后启用
ln -sf /etc/nginx/sites-available/safemask-auth /etc/nginx/sites-enabled/
nginx -t
nginx -s reload
```

### 3. （可选）Nginx 层频率限制

在 `/etc/nginx/nginx.conf` 的 `http {}` 中添加：

```nginx
limit_req_zone $binary_remote_addr zone=safemask_dl:5m rate=1r/s;
limit_conn_zone $binary_remote_addr zone=conndl:5m;
```

然后在 location 中取消注释相应行。建议先观察正常流量后再按需启用。

## 分步操作（等效于 deploy.sh）

### 生成 HMAC 密钥

```bash
sudo bash /opt/safemask-auth/init-key.sh                    # 首次
sudo bash /opt/safemask-auth/init-key.sh --force             # 重新生成
bash /opt/safemask-auth/init-key.sh --dry-run                # 模拟
```

密钥保存在 `/opt/safemask-auth/config/hmac.key`（chmod 600）。

### 手动启动服务

```bash
export HMAC_KEY=$(cat /opt/safemask-auth/config/hmac.key)
export DOWNLOAD_URL="https://obs.be.com:9004/gxzh/2026/07/06/privacy-filter.zip"
pm2 start /opt/safemask-auth/server.js --name safemask-dl-auth --update-env
pm2 save
```

### 验证部署

```bash
BASE="https://www.xiao.org.cn"
curl -i "$BASE/health"                                          # 期望 200
curl -i "$BASE/download"                                         # 期望 400
curl -i "$BASE/"                                                 # 期望 404
curl -i "$BASE/download?token=invalid:device:0"                  # 期望 403

# 有效 token
HMAC_KEY=$(cat /opt/safemask-auth/config/hmac.key)
DEVICE_ID="test-verify-$(date +%s)"
TIMESTAMP=$(date +%s)
SIG=$(echo -n "$DEVICE_ID:$TIMESTAMP" | openssl dgst -sha256 -hmac "$HMAC_KEY" -binary | base64 -w0 | tr '+/' '-_' | tr -d '=')
TOKEN="${SIG}:${DEVICE_ID}:${TIMESTAMP}"
curl -i "$BASE/download?token=$TOKEN"                             # 期望 302
curl -i "$BASE/download?token=$TOKEN"                             # 期望 429
```

## 日常运维

```bash
pm2 logs safemask-dl-auth       # 实时日志
pm2 monit                       # 资源监控
pm2 restart safemask-dl-auth    # 重启
pm2 stop safemask-dl-auth       # 停用

# 密钥轮换
sudo bash /opt/safemask-auth/init-key.sh --force
sudo bash /opt/safemask-auth/deploy.sh --restart-only
# 同时本地用新密钥重新构建 Rust 二进制
```

## 文件清单

| 文件 | 作用 |
|------|------|
| `server.js` | 鉴权服务（零依赖 Node.js 内置模块） |
| `init-key.sh` | HMAC 密钥生成与持久化 |
| `deploy.sh` | 一键部署脚本（管理密钥 + PM2 + 验证） |
| `nginx.conf.template` | Nginx 配置参考模板（手动配置） |
| `/opt/safemask-auth/config/hmac.key` | 生成的密钥文件（chmod 600） |
| `/opt/safemask-auth/config/safemask.env` | 环境变量文件 |
